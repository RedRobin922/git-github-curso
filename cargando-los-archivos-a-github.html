Cargando Los Archivos de un Repositorio Local a Github

Subir un repositorio a Github es quiza, una de las practicas mas importantes del proceso de desarrollo. Ya que esto nos permite tener un respaldo en la nube de cualquier proyecto que estemos llevando a cabo.

  • Primer Paso: Creando un Repositorio Remoto

    • Seleccionamos el boton "create repository", esto nos llevara a la ventana de creacion de repositorio. En la opcion "repository name", colocaremos el nombre del repositorio del proyecto en el que estamos trabajando. La opcion de descripcion es completamente opcional, pero tampoco es algo que este de sobra, el dejar una descripcion del repositorio, esto es especialmente util si nuestro repositorio sera publico. Una vez concluidos estos pasos, se selecciona la opcion "create repository" y ya tendremos nuestro repositorio remoto creado.

  • Primer Push a Github
    • Una vez creado el repositorio, procedemos a hacer nuestro primer push. Que es un push? Pues tan simple como el 'subir' toda la informacion que contiene el repositorio de Git a nuestro repositorio, o respaldo, remoto en Github. Para esto introducimos el siguiente comando en Git Bash: git remote add origin 'direccion de tu repositorio en github'. El espacio enmarcado como 'direccion de tu repositorio en Github' te lo provee la propia pagina en la seccion "Quick setup", que suele aparecer inmediatamente despues de crear el repositorio remoto. Esta direccion personal se ve de la siguiente manera: "https://github.com/tuusuario/elnombredeturepositorio".
