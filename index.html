Git - Conceptos Basicos

Git se subdivide en tres areas basicas:

  • Directorio de Trabajo

    • Es donde se crean los archivos en la memoria de Git antes de hacer el guardado

  • Area de Ensayo

    • Es un area que almacena los archivos de manera temporal, aca se puede ver que archivos tienen seguimientos y cuales no, asi como el estado donde esta cada archivo

  • Repositorio Local

    • Es donde se almacenan las instantaneas que toma Git de nuestros archivos y que serviran para hacer recuperacion posterior de los cambios en los archivos

Primeros Pasos en Git

  • Git Init

    • Se introduce en la consola, y este comando (que se ejecuta solo una vez), le indica a Git que debe comenzar a llevar un seguimiento de los cambios del proyecto

  • Git Add

    • Este comando le indica a Git que el archivo seleccionado debe ser enviado al area de ensayo, esto hace que Git comience a llevar registro de los cambios del archivo

  • Git Commit

    • El archivo es enviado al repositorio local, en este momento es cuando se hace una diapositiva de los ultimos cambios del archivo, y es el registro que se cargara para trabajos posteriores

Creando un Repositorio

  • Git Init

    • Este, como ya mencionamos anteriormente, es el primer paso. Pero para esto, te vas hacia el directorio en tu PC o servidor donde tienes guardadp el archivo HTML y con click derecho, seleccionas la opcion "Bash Here". Esta opcion activa automaticamente el Git Bash dentro del directorio seleccionado. Otro metodo largo (tambien en Windows) es el de seleccionar directamente Git Bash, abrir la consola de comandos e introducir los comandos uno por uno hasta llegar al directorio. Una vez alli, tipeas "Git Init", para iniciar el proceso de Git

  • Staging Area

    • Al comenzar a trabajar en un documento dentro del directorio que esta siendo respaldado, y cuando se quiera 'crear un checkpoint de los cambios', asi como para que Git sepa que el archivo tiene que ser monitoreado para registrar los cambios, se debe enviar al area de Stage. Para esto, utilizamos el comando git add 'nombre del archivo'.

  • Git Status

    • Permite ver el estado de los archivos con los que estamos trabajando en el directorio que esta siendo guardado en Git. A - Significa que el archivo se encuentra en la staging area. Para dar inicio al git status, el comando es "git status -s".

  • Git Commit

    • Es el paso final en el repositorio Git, este comando envia el archivo desde la Staging Area al Repositorio Local. El comando de activacion es "git commit -m". Se acompa;a con un nombre para el Commit o para el respaldo, colocado entre comillas despues de la 'M'. Antes de hacer el commit, se debe configurar un correo y un nombre dentro de Git Hub. Para esto se utilizan los siguientes comandos: - git config --global user.name "tu nombre de usuario" (para registrar el nombre de usuario). - git config --global user.email "tu direccion de correo electronico" (para registrar el correo electronico).

Como se Que Mi Archivo Esta Sincronizado con el Repositorio?

Utilizamos el comando 'git status -s'. Una vez que el archivo se encuentra incluido en el respositorio, desaparecera de la lista en el Git Bash.

Que Pasa si Hago un Cambio Despues de Enviar un Commit a un Repositorio?

Inmediatamente despues de hacer cualquier cambio en un archivo, debemos enviar los archivos al area de Stage (git add 'nombre del archivo') Si nos saltamos este paso, Git no sera capaz de reconocer que el archivo esta siendo editado, y cuando se quiera hacer un 'commit', arrojara el error de 'Changes not staged for commit'.

Como Saber Cuantas 'copias' y/o 'commit' se han hecho en el Repositorio Local?

git log --oneline Este comando nos sirve para obtener un listado de todos los cambios que se han registrado en el repositorio local.
